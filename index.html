Arrow UDFs in action

# before: Pandas UDF - converts every batch to pandas and back
@pandas_udf("double")
def to_celsius(f: pd.Series) -> pd.Series:
    return (f - 32) * 5 / 9

# after: native Arrow UDF - no conversion tax
from pyspark.sql.functions import arrow_udf
import pyarrow as pa, pyarrow.compute as pc

@arrow_udf("double")
def to_celsius(f: pa.Array) -> pa.Array:
    return pc.multiply(pc.subtract(f, 32), 5 / 9)

# aggregates work too: pa.Array in, scalar out
@arrow_udf("double")
def weighted_mean(v: pa.Array, w: pa.Array) -> float:
    return pc.sum(pc.multiply(v, w)).as_py() / pc.sum(w).as_py()

NEAREST BY: top-K ranking join

Today — the pain
  • Top-K per query row means a cross join, a ROW_NUMBER() window, a rank, then a filter
  • The optimizer can’t see the intent — so it can’t prune or speed it up
  • Swapping in an index later means rewriting the query
Spark 4.2 — the mechanism

Nanosecond-precision timestamps

  • SPIP SPARK-56822: timestamps gain explicit fractional-second precision — today Spark stops at microseconds
  • Nanosecond Parquet (TIMESTAMP(NANOS)) currently errors out, or falls back to raw LongType — losing time-zone and timestamp semantics
  • New parameterized types TIMESTAMP_NTZ(n), TIMESTAMP_LTZ(n), TIMESTAMP(n) with n in [0, 9] (6 = micros, 9 = nanos); focus on 7–9
